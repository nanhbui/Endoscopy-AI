Why _ws_lock? Starlette WebSocket raises if 2 coroutines write concurrently. _relay_events + _stream_llm can both fire send_json at the same time (LLM streams while pipeline still emits state events). Lock serializes all writes.

Pipeline = Subprocess (spawn). Tại sao?

3 conflicts ép buộc isolation

  • CUDA + fork = đứng máy — PyTorch/Ultralytics khởi động CUDA context không thể fork an toàn
  • GLib threads ≠ uvloop — GStreamer dùng GMainLoop riêng, conflict với asyncio loop của FastAPI
  • Blocking inference — YOLO call ~50–200ms blocks event loop nếu chạy chung process
_mp_ctx = mp.get_context("spawn") — spawn (không fork) tạo Python interpreter mới hoàn toàn.

IPC contract

  • result_q (worker → main) — events
  • cmd_q (main → worker) — STOP / RESUME / IGNORE:<fi>:<json>
  • _bridge_loop (daemon thread) — đẩy result_q → asyncio.Queue (events)
self._proc = _mp_ctx.Process(
    target=_pipeline_worker,
    args=(video_path, model_path, conf,
          result_q, cmd_q, reid_weights,
          gstshark_*),
    daemon=True,
)

Design Decision: Spatial-Temporal Dedup

Old approach (broken): _track_reported: set — once track_id reported, suppressed forever. Combined with max_age=300, the tracker re-associated returning lesions with old IDs → silently dropped detections in second half of the video.

New approach

_DEDUP_WINDOW_MS = 10_000   # 10 s
_DEDUP_IOU       = 0.25
_reported_history: list[dict] = []

def _is_recently_reported(ts_ms, bbox, label):
    for r in _reported_history:
        if (ts_ms - r["ts_ms"] < _DEDUP_WINDOW_MS
            and r["label"] == label
            and _iou(r["bbox"], bbox) >= _DEDUP_IOU):
            return True
    return False

Why it works

  • Same lesion within 10 s → suppressed (avoid spam)
  • Different lesion same frame → reported (different bbox)
  • Same lesion 30 s later → reported again (doctor có thể muốn xem lại pass khác)
  • Tracker-agnostic — không phụ thuộc track ID stability; works even if tracker disabled
Trade-off: O(N) lookup per detection. N = số detection trong session, thường < 100 → negligible.

LLM System Prompt — Paris Classification

## Hệ thống phân loại Paris (bắt buộc áp dụng)

**Dạng polypoid (nhô cao):**
- 0-Ip (cuống): Tổn thương có cuống rõ ràng
- 0-Is (không cuống): Nhô cao, đáy rộng

**Dạng phẳng/không polypoid:**
- 0-IIa (nhô thấp): < 2.5mm
- 0-IIb (phẳng hoàn toàn): màu sắc thay đổi
- 0-IIc (lõm nông): < 1.2mm
- 0-IIa+IIc (kết hợp): nguy cơ xâm lấn cao

**Dạng lõm sâu:**
- 0-III (loét): > 1.2mm, bờ fibrin

## Format phản hồi (bắt buộc 3 sections)
**Phân loại Paris:** [Loại] — [mô tả màu sắc, bờ, kích thước]
**Nhận định lâm sàng:** [lành / tiền ung thư / ác tính] — [lý do]
**Checklist hành động:**
- [ ] Bước 1
- [ ] Bước 2
...
Why structured prompt? Endoscopy reporting cần Paris classification chính xác (FR-013 baseline). Format 3 sections giúp UI render markdown đẹp + giảm hallucination. Prompt cố định ~1500 tokens → OpenAI auto-cache sau request đầu.

Why virtual 1920×1080?

  • FE constants FRAME_W/H hard-coded
  • Source video resolution varies (720p, 1080p, 4K)
  • Single normalization step in BE → FE math always correct

Bbox label flip

If bbox.y < 6% (lesion ở mép trên), label flips từ top: -36px (above) sang top: 4px (inside) — tránh bị crop khỏi video container.

Failure Modes & Safeguards

FailureSymptomFix
WS concurrent writes Starlette raises, WS closes, file deleted asyncio.Lock serializes send_json
STOP race vs EOS event Worker return, EOS skipped, FE stuck should_stop=True; break → finally emits _eos_event
Worker crash (OOM, segfault) Bridge sees dead proc + empty queue, exits silently Bridge synthesizes VIDEO_FINISHED with accumulated detections
Track dedup blocks 2nd half Lesions returning to same area never fire again Spatial-temporal dedup (10s window, IoU 0.25)
Bbox label clipped at top Label overflows above video container Auto-flip when bbox.y < 6%
"Session not found" race WS reconnect after upload deleted FE handler resets state to IDLE on this specific error
Ignore button frozen FE stays PAUSED after "Bỏ qua" Optimistic UI update + _set_state(PLAYING) in worker after IGNORE
Library video deletion blocked Lingering session locks file _session_active() checks ctrl._proc.is_alive(), not just registry
